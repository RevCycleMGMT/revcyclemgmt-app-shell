# Security

This is a public synthetic app-shell demo.

Do not commit PHI, production claims, payer files, payment files, credentials, cloud secrets, database dumps, screenshots with identifiers, client extracts, or production source-system exports.

The local server is for proof review only. It has no authentication layer, no persistence, no upload endpoint, and no production data path. Production work would require a separate security design with role-based access control, audit logging, retention rules, vendor review, and formal agreements where appropriate.
